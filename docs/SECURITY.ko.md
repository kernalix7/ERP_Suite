## 취약점 제보 방법

**보안 취약점은 공개 GitHub Issue로 제보하지 말아 주세요.**

대신 [GitHub Security Advisories](https://github.com/kernalix7/ERP_Suite/security/advisories/new)로 제보해 주세요.

### 제보 시 포함할 내용

1. **설명** — 취약점에 대한 명확한 설명
2. **재현 방법** — 상세한 재현 절차
3. **영향도** — 예상 가능한 영향 (예: 데이터 노출, 권한 상승)
4. **영향 범위** — 영향받는 구성요소 (예: 인증, API, 파일 업로드)
5. **환경 정보** — OS, Python 버전, Django 버전, 배포 방식

### 대응 일정

- **접수 확인** — 제보 후 48시간 이내
- **초기 평가** — 7일 이내
- **수정 및 공개** — 제보자와 조율하여 진행 (치명적 이슈는 통상 30일 이내)

### 범위 (In Scope)

다음 항목은 보안 제보 범위에 포함됩니다:

- 인증/인가 우회 (RBAC, JWT)
- SQL Injection, XSS, CSRF 취약점
- 민감 데이터 노출 (API 키, 자격 증명, 개인정보)
- 파일 업로드 취약점 (경로 탐색, 무제한 업로드)
- 세션 관리 이슈
- 안전하지 않은 직접 객체 참조 (IDOR)
- 서버 사이드 요청 위조 (SSRF)
- 역할 간 권한 상승 (직원/매니저/관리자)

### 범위 외 (Out of Scope)

- 서버에 물리적으로 접근해야만 가능한 문제
- 사회공학 기반 공격
- 서드파티 의존성 자체 이슈 (가능하면 upstream에 제보, 필요 시 본 프로젝트에도 공유)
- 무차별 대입을 통한 서비스 거부 (django-axes로 완화됨)

## 보안 모범 사례

ERP Suite는 다음 보안 원칙을 따릅니다:

- **RBAC 적용** — 3단계 역할 시스템 + Mixin 기반 접근 제어
- **무차별 대입 방지** — django-axes (5회 실패 시 계정 잠금)
- **원자적 연산** — F() 표현식으로 경쟁조건 방지 재고 업데이트
- **감사 추적** — django-simple-history (전 모델) + AccessLogMiddleware
- **입력 검증** — 파일 업로드 화이트리스트, 10MB 제한, 폼 유효성 검사
- **운영 보안** — HSTS, SSL 리다이렉트, HttpOnly 쿠키, 8시간 세션 만료

## 감사의 말

책임 있는 공개(Responsible Disclosure)에 기여해 주는 보안 연구자 커뮤니티에 감사드립니다. 유효한 제보자는 동의하에 릴리스 노트에 기여자로 표기할 수 있습니다.
